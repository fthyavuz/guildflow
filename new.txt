Strategic Vision & Technical Objectives Document1. Primary MissionThe aim of this system is to bridge the gap between spiritual goals and daily habits by providing a structured, verifiable, and visually motivating tracking platform. It transforms abstract spiritual milestones (e.g., memorization, consistent prayer, reading) into measurable data points that allow for meaningful mentorship between Mentors and Students.2. Structural Objectives: The "Template-to-Action" ModelOur aim is to create a highly modular system where content is separated from scheduling. This is achieved through three distinct layers:The Resource Library (Global Inventory): To create a standardized database of tasks. Whether it is a 600-page Quran, a 30-minute podcast, or a binary "Memorized/Not Memorized" Surah, every task is defined by its Total Capacity and Category.The Checklist (Grouping Logic): To allow Mentors to bundle various resources into "Checklist Templates" (e.g., "Ramadan Program" or "Monthly Foundation Course") without being tied to specific dates yet.The Assignment (Live Implementation): To turn a template into a living task for a specific Class or Student by defining the Start/End Dates and the Frequency (Daily/Weekly).3. User Experience (UX) Goals: Mobile-First Data EntryA critical aim is to ensure high engagement by removing friction in the data entry process.The "Period-First" Interface: Recognizing that most students use mobile devices, we aim to move away from complex 2D tables. The system must present a Date/Period Selector first, followed by a simple Vertical Task List.Real-Time Feedback: As a student enters data for a specific day, the UI must immediately display a Total Summary at the bottom of the form.Objective: The student should see: "I entered 5 pages today; my total is now 150/300 (50%)." This provides instant gratification and clarity.4. Integrity & Validation ObjectivesTo maintain the value of the spiritual progress, the data must be reliable.The Approval Workflow: No data is "official" until it is reviewed. The aim is a Staged Entry system:Student Input $\rightarrow$ 2. Pending Status $\rightarrow$ 3. Mentor Review/Approval $\rightarrow$ 4. Report Card Update.Logical Constraints: The system must prevent "impossible" data entries. If a resource has a Daily Limit, the student cannot enter a higher value, and they can never exceed the Total Capacity of the resource.5. The Final Output: The Growth-Oriented Report CardThe ultimate goal of the data collection is the Student Report Card.Read-Only Transparency: Both Parents and Students have read-only access to this dashboard.Visual Progress: The aim is to convert numbers into Percentage-Based Visuals (Progress bars, circular charts).Binary vs. Linear Tracking: * For Measurable tasks (Books/Podcasts), progress is a linear percentage.For Memorization tasks, the aim is a "Milestone" approach: $0\%$ until the Mentor confirms completion, then a jump to $100\%$.6. Administrative ScalabilityFor the Admin Panel, the aim is Dynamic Configuration:The Admin must be able to add new Sub-Categories (e.g., adding "Social Service" or "Physical Exercise") without any code changes.This ensures the application can evolve beyond its current scope and adapt to different educational or spiritual curriculums.Summary for Development AgentsTechnical Goal: Build a system where Assignments act as the bridge between Users and Checklist Templates, calculating Intervals dynamically based on date ranges, and aggregating Approved Inputs to render a real-time Report Card.
